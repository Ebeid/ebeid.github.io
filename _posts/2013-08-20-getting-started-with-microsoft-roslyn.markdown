<p>In a previous <a href="http://ebeid-soliman.blogspot.com/2013/08/overview-of-microsoft-roslyn-first.html" target="_blank">post</a> we talked about Microsoft <a title="Microsoft&reg; &ldquo;Roslyn&rdquo; CTP" href="http://msdn.microsoft.com/en-us/vstudio/roslyn.aspx" target="_blank">Roslyn</a>. In this post will get our hands dirty with Roslyn syntax analysis in order to develop our first code-focused program. We will see how the SyntaxTree of HelloWorld program look like and how we can traverse and query it.</p> <p>The<b> Syntax API</b> exposes the syntax trees the compilers use to understand Visual Basic and C# programs. They are produced by the same parser that runs when a project is built or a developer hits F5. The four primary building blocks of syntax trees are:</p> <ul> <li>The <b>SyntaxTree</b> class, an instance of which represents an entire parse tree.  <li>The <b>SyntaxNode</b> class, instances of which represent syntactic constructs such as declarations, statements, clauses, and expressions.  <li>The <b>SyntaxToken</b> structure, which represents an individual keyword, identifier, operator, or punctuation.  <li>And lastly the <b>SyntaxTrivia</b> structure, which represents syntactically insignificant bits of information such as the whitespace between tokens, preprocessor directives, and comments.</li></ul> <h3>Traversing Syntax Trees</h3> <p>First, download Microsoft Roslyn CTP from <a title="Microsoft Download Center" href="http://www.microsoft.com/en-us/download/details.aspx?id=34685" target="_blank">here</a>. and Install it.You will need <a href="http://go.microsoft.com/fwlink/?linkid=255983" target="_blank">Visual Studio 2012</a> and <a href="http://go.microsoft.com/?linkid=9810482" target="_blank">Visual Studio 2012 SDK</a> in order to install Roslyn.</p> <p>Now open your Visual Studio 2012 and create a new Console Application project. Add the following to your project references</p> <h3><a href="http://lh6.ggpht.com/-UBd4AZvbDW4/UhKWobk-NBI/AAAAAAAAB3o/CQLxCuBSrlA/s1600-h/Reference%252520Manager%252520-%252520RoslynDemo1_2013-08-19_15-25-47%25255B3%25255D.png"><img style="background-image: none; First, we included the needed namespaces. The common Syntax APIs are found in the Roslyn.Compilers and the Roslyn.Compilers.Common namespace, while the language specific Syntax APIs are found in Roslyn.Compilers.CSharp and Roslyn.Compilers.VisualBasic.</p><br /><p>Second, we tried to get a syntax tree corresponding to our HelloWorld program using the <em>SyntaxTree.ParseText</em> function, which produces a syntax tree by parsing the passed source code.</p><br /><p>Third, in order to traverse the syntax tree we get its root, then get its child nodes. Then loop on these child nodes and their children recursively to print its kind and the string representation. C# node kind is on of the kinds in enum <em>Roslyn.Compilers.CSharp.SymbolKind</em>.</p><br /><p>Inside the PrintSyntaxTree method, by playing in the order of node printing and the recursive call, you can apply your tree traversal logic. You can also inspect specific elements of the tree:</p><pre class="prettyprint lang-cs">var root = (CompilationUnitSyntax)tree.GetRoot(); <br />var helloWorldDeclaration = (NamespaceDeclarationSyntax)root.Members[0];<br />var programDeclaration = (TypeDeclarationSyntax)helloWorldDeclaration.Members[0];<br />var mainDeclaration = (MethodDeclarationSyntax)programDeclaration.Members[0];<br />var argsParameter = mainDeclaration.ParameterList.Parameters[0];<br />Console.WriteLine(argsParameter.ToString() );<br /></pre><br /><h3>Query Methods</h3><br /><p>In addition to traversing syntax trees you can also explore the syntax tree using the query methods defined on <em>SyntaxNode</em>.     where methodDeclaration.Identifier.ValueText == "Main"<br />                                  select methodDeclaration.ParameterList.Parameters.First();<br /><br />            var argsParameter2 =
